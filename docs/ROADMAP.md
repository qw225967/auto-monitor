# Auto Monitor Roadmap

## 目标

- 以真实可执行套利为目标，降低无效机会与误报。
- 在 CoinGecko 月度限额下建立稳定的请求预算与可观测性。
- 从“机会发现”升级为“净收益排序 + 执行准备”。

## 当前阶段结论

- 当前主链路已具备：机会计算、通路探测、流动性过滤、前端展示。
- 关键缺口：请求预算管理、净收益模型、路径质量评分、执行前约束。

## 里程碑

### M1: 稳定性与配额安全 (P0/P1)

- 修复 CoinGecko 鉴权 header 逻辑（Demo/Pro 一致化）。
- TokenSync 从“资产级跳过”改为“链级增量 + TTL 刷新”。
- LiquiditySync 增加 retry/backoff/jitter。
- Network registry 刷新改为“局部更新 + TTL”，避免一次失败清空缓存。
- RunDetect 增加并发上限与超时分层。

验收标准：
- CoinGecko 429 比例 < 2%。
- 由缓存抖空导致的可用路径突降事件为 0。
- 探测周期稳定在目标窗口内（P95 < 30s，按运行环境可调）。

### M2: 请求价值化调度

- 引入 CoinGecko 预算管理器（全局日/月预算）。
- 引入负缓存（404/无池/不支持链）。
- 引入分层 TTL（高价值短 TTL，低价值长 TTL）。
- 由全表流动性同步改为“候选优先 + 背景补全”。

验收标准：
- 月请求量较基线下降 >= 40%。
- 有效请求命中率（返回可用流动性）提升 >= 30%。
- Top 候选数据新鲜度满足目标（例如 15 分钟内）。

### M3: 机会质量升级

- 增加净收益模型：交易费、提现费、桥费、滑点、时间风险。
- 输出字段升级：gross_spread / estimated_cost / net_spread / confidence。
- 引入路径质量评分（成功率、时延、稳定性）。

验收标准：
- 高优先候选的“可执行命中率”显著提升。
- 假阳性（无法执行机会）占比持续下降。

### M4: 执行就绪

- 接入资金与风控约束：余额、最小下单量、提现限额、维护状态。
- 形成“监控候选榜单”和“可执行候选榜单”双视图。

验收标准：
- 候选到可执行的转化率稳定可观测。
- 核心流程支持灰度启用与回滚。

## 依赖与风险

- 外部 API 速率限制和策略变更（CoinGecko、交易所）。
- 链上/交易所状态波动导致的数据抖动。
- 历史数据不足时，评分模型初期偏差较大。

## 非目标（当前阶段）

- 不在 M1 阶段实现全自动实盘执行。
- 不在 M1 阶段引入复杂机器学习模型。
